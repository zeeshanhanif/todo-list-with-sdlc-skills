# Design System: To-Do List Application

> Source of truth for all UI implementation, with `docs/tokens.json`.
> Generated by ux-foundations · Mode: research (designed from requirements) · Direction: "Fresh Calm" (teal + amber)
> Component framework: **shadcn/ui** (Radix primitives + Tailwind) · Icons: **Lucide**
> Last updated: 2026-07-15

## 1. Visual Theme & Atmosphere

Fresh, friendly, and calm — a productivity tool that feels light and gets out of
the way. Clean cool-white surfaces, generous but adaptive spacing, a confident
teal that signals action and completion, and warm amber for small moments of
emphasis. Rounded corners and soft shadows keep it approachable rather than
corporate; the copy is playful (see §6) while the layout stays quiet so tasks are
the focus. Density is **adaptive**: comfortable and touch-friendly on mobile,
a little denser on wide desktop viewports. Full light and dark themes are
first-class (theme is a user setting, FR-PROF-004).

## 2. Design Tokens

Semantic, role-based. **Derived from `docs/tokens.json` — same values, embedded
here as CSS custom properties for direct use.** Never hardcode a raw value that a
token exists for.

### Spacing, Radius, Elevation

- **Spacing (4px base):** page gutters 16–24px; section gap 32px; card padding
  16–20px; list-row vertical padding 8–12px (see adaptive density, §3);
  intra-component gap 8–12px. Pick spacing **only** from `--space-*`.
- **Radius:** controls/buttons/inputs `--radius-md` (8px); cards/panels/dialogs
  `--radius-lg` (12px); checkboxes/avatars/priority dots/pills `--radius-full`.
- **Elevation:** `--shadow-sm` for resting cards/rows; `--shadow-md` for popovers,
  dropdowns, the task-detail panel; `--shadow-lg` for modals/dialogs. On dark,
  lean on borders plus the deepened shadows.

## 3. Layout & Grid

- **Breakpoints:** `sm 640` · `md 768` · `lg 1024` · `xl 1280`. Mobile-first.
- **App shell (authenticated):**
  - **≥ lg:** persistent left **sidebar** (`--sidebar-width` 280px) + main content
    column (`--content-max` 760px, centered) + optional right **task-detail panel**
    (`--detail-panel-width` 420px) that slides in.
  - **md:** sidebar collapses to an icon rail or a toggled drawer; detail opens as
    an overlay panel.
  - **< md (mobile):** sidebar becomes a full drawer (hamburger); main column is
    full-width; task detail opens **full-screen**; a bottom-anchored quick-add.
- **Auth screens:** single centered card, max-width 400px, on the app background.
- **Adaptive density:** `--control-md` (40px) rows/controls are the default;
  **≥ lg**, list rows may compact toward `--control-sm` (32–36px) and tighten
  vertical padding to `--space-2`. **< md**, every interactive target is
  `--touch-target` (44px) minimum. Density never drops below the touch minimum on
  touch viewports.

## 4. Components

Built with **shadcn/ui** (Radix + Tailwind). Map each spec below to its shadcn
component and theme it with the tokens above; do not fork a parallel system.
Every interactive component must implement **default / hover / focus-visible /
active / disabled** (plus **loading / error** where noted).

### Core — Actions
- **`button-primary`** — `--color-primary` fill, `--color-on-primary` text,
  radius `--radius-md`, height `--control-md` (48px prominent / touch), padding
  `--space-3` × `--space-5`, weight 500. Hover `--color-primary-hover`; active
  `--color-primary-active`; focus-visible 2px `--color-focus-ring` ring + 2px
  offset; disabled 45% opacity, `cursor: not-allowed`; loading shows a spinner and
  disables. Maps to shadcn `Button` (default variant).
- **`button-secondary`** — `--color-surface` fill, `--color-border-strong` 1px
  border, `--color-text` text. Hover `--color-surface-sunken`. (shadcn `outline`.)
- **`button-tertiary` / ghost** — transparent, `--color-primary` text, no border;
  hover `--color-primary-subtle` bg. (shadcn `ghost`.)
- **`button-danger`** — `--color-danger` fill, white text; hover
  `--color-danger-hover`. For confirmed destructive actions only. (shadcn
  `destructive`.)
- **`icon-button`** — 40px (44px touch) square, transparent; hover
  `--color-surface-sunken`; same focus ring. Always has an `aria-label`.
- **`link`** — `--color-primary` text, underline on hover/focus.

### Core — Forms
- **`input` / `textarea` / `select`** — `--color-surface` bg, 1px
  `--color-border-strong`, radius `--radius-md`, height `--control-md`, padding
  `--space-3`; text `--color-text`, placeholder `--color-text-subtle`. Focus:
  border `--color-primary` + 2px `--color-focus-ring` ring. Error: border
  `--color-danger`, error text below in `--color-danger` `small`. Disabled: sunken
  bg, muted text. (shadcn `Input`/`Textarea`/`Select`.)
- **`checkbox`** — the **task-complete** control: `--radius-full` circle, 20px
  (44px hit area), 2px `--color-border-strong` unchecked; checked = `--color-success`
  fill with white check; focus ring as above; animates check in `--duration-fast`.
- **`radio` / `switch`** — Radix primitives themed with `--color-primary` when on.
- **`field`** — label (`caption`, `--color-text`), optional help (`small`,
  `--color-text-muted`), error slot (`small`, `--color-danger`). Label always
  present and associated (`for`/`id`).
- **Validation convention:** validate on blur and on submit; show one error per
  field beneath it; summarize form-level errors in an inline alert at the top.

### Core — Feedback
- **`toast`** — bottom-center (mobile) / bottom-right (desktop), `--color-surface`,
  `--shadow-lg`, radius `--radius-lg`, auto-dismiss ~5s. Variants: neutral,
  success, danger. The **undo snackbar** (task delete) is a toast with an "Undo"
  `button-tertiary` and a ~7s timeout matching the restore window.
- **`inline-alert`** — subtle tinted bg (`--color-*-subtle`) + matching text token,
  icon, radius `--radius-md`. For form errors and page-level notices.
- **`dialog` / `modal`** — centered card, radius `--radius-lg`, `--shadow-lg`,
  `--color-overlay` scrim; focus-trapped; Esc + scrim-click close (except blocking
  confirmations). (shadcn `Dialog`.)
- **`confirm-dialog`** — title, body, `button-secondary` cancel + `button-danger`
  confirm. Used for delete list, delete account, delete task-with-warning
  (destructive-action rule, §A5 / NFR-USE-002).
- **`spinner` / `skeleton`** — skeleton rows for list/detail loading; spinner for
  in-button and short waits.

### Core — Structure & Navigation primitives
- **`card`** — `--color-surface`, 1px `--color-border`, radius `--radius-lg`,
  padding `--space-4`–`--space-5`, `--shadow-sm`.
- **`list` / `list-row`** — see task-row (§ surface). Generic rows: 1px bottom
  `--color-border`; hover `--color-surface-sunken`.
- **`tabs`**, **`dropdown-menu`**, **`tooltip`**, **`popover`** — Radix/shadcn,
  themed; popovers use `--shadow-md`.
- **`badge` / `chip`** — `caption`, `--radius-full`, tinted bg + matching text
  token (e.g., due-soon uses `--color-warning-subtle` + `--color-warning-text`).
- **`avatar`** — `--radius-full`, initials on `--color-primary-subtle`.

### Data-view state conventions (define once, use everywhere)
- **Loading:** skeleton rows (not a blank screen).
- **Empty:** centered illustration/icon + `h3` headline + one-line playful
  subtext + a primary action (e.g., "Add your first task"). Distinct copy per
  context (empty list vs. empty smart view vs. no search results).
- **Error:** inline-alert with a plain explanation and a retry action; preserve
  any unsent user input (NFR-REL-004).

### Surface-specific components (Web) — beyond the core
- **`app-shell`** — sidebar + main (+ detail panel) responsive frame (§3).
- **`sidebar-nav-item`** — icon + label + optional count badge; selected state uses
  `--color-primary-subtle` bg + `--color-primary` text + left accent bar. Used for
  smart views and lists.
- **`task-row`** — complete-`checkbox` + title (`body`, strikethrough + muted when
  completed) + right cluster: due-date `chip` (turns `--color-danger` when
  overdue), priority dot (`--color-priority-*`), and a drag handle (`≥lg`, for
  reorder). Full row is the click target for detail; 44px min height on touch.
- **`quick-add`** — persistent single-line composer ("Add a task…") with inline
  affordances to set due date and priority; Enter creates. Bottom-anchored on
  mobile.
- **`task-detail-panel`** — right slide-in (`≥lg`) / full-screen (mobile): editable
  title, list picker, due-date picker, priority selector, complete + delete.
- **`due-date-picker`** — popover calendar + time; clearable; interprets the user's
  timezone (FR-PROF-003).
- **`priority-selector`** — segmented/dropdown of None/Low/Medium/High with the
  priority colors.
- **`list-picker`** — dropdown of the user's lists (for task's list at create).
- **`command-search`** — search field opening an overlay of results grouped by list.

## 5. Accessibility Rules

Standard: **WCAG 2.2 AA as the working guideline** for contrast, focus, and
targets. Per **NFR-USE-004** the MVP commits to accessibility *best practices*
rather than formal AA certification, but this system is built to AA so the option
stays open (see Open Questions). Also realizes **NFR-USE-002** (confirm
destructive actions) and **NFR-USE-003** (loading/empty/error states).

- **Contrast:** body text ≥ 4.5:1, large text (≥ 24px or 18.66px bold) and UI
  graphics ≥ 3:1. All token pairings above are chosen to meet this; never place
  body text in `--color-text-subtle`, raw `--color-primary-bright`, raw
  `--color-accent`, or `--color-success` fill.
- **Focus:** every interactive element shows a visible 2px `--color-focus-ring`
  ring with 2px offset on `:focus-visible`. Never remove outlines without a
  replacement.
- **Targets:** ≥ 44×44px on touch viewports (`--touch-target`); the compact
  desktop density (§3) applies only with a pointer.
- **Keyboard:** full operability — tab order follows reading order; Enter creates a
  task; Space toggles complete; Esc closes panels/dialogs; arrow keys move within
  lists; drag-reorder has a keyboard alternative (move up/down). Focus is trapped
  in dialogs and restored on close.
- **Semantics:** real landmarks (`nav`, `main`), lists as `ul`/`li`, buttons as
  `button`, labels associated with inputs, `aria-live` for toasts and async
  results, icon-only controls have `aria-label`.
- **Motion:** respect `prefers-reduced-motion` (CSS above); no essential
  information conveyed by motion or color alone (priority also shows a label in
  the detail view, overdue shows text not just red).

## 6. Voice & Microcopy

Voice: **playful, warm, and brief** — human, never cutesy to the point of
friction. Tone flexes down to plain and reassuring for errors and destructive
actions (never joke in a delete confirmation or an error).

- **Do:** "Inbox zero. Go enjoy your day. 🎉" · **Don't:** "You have 0 remaining
  task items."
- **Do (empty):** "Nothing here yet — add your first task and get rolling." ·
  **Don't:** "No data available."
- **Do (error, plain):** "Couldn't save that just now. We kept your changes —
  try again." · **Don't:** "Error 500: request failed."
- **Do (destructive, serious):** "Delete this list and its 12 tasks? This can't be
  undone." · **Don't:** "Whoops, bye-bye list! 👋"
- Buttons are verbs ("Add task", "Save changes", "Delete account"). Sentence case
  everywhere. Dates human-friendly ("Today", "Tomorrow", "Fri", "Mar 3").

## 7. Do's and Don'ts

- **Do** use tokens for every color, space, radius, shadow — never raw hex/px.
- **Do** pick spacing only from `--space-*` and type only from the scale.
- **Do** implement all five interactive states + loading/empty/error for data views.
- **Do** build on shadcn/ui components, themed via tokens; keep Lucide for icons.
- **Do** keep the accessible action tokens (`--color-primary`, `--color-*-text`)
  for anything legible; reserve bright brand hues for non-text accents.
- **Don't** put body text on `--color-text-subtle` or raw teal/amber.
- **Don't** invent component variants — extend via the defined axes/tokens.
- **Don't** convey meaning by color alone (pair with text/icon).
- **Don't** remove focus outlines; don't drop below 44px targets on touch.
- **Don't** restate token values in `ux-foundations.md` — this file is the source.

- **Destructive** → `confirm-dialog` + `button-danger`.
- **Most-used tokens:** text `#0F172A` / muted `#64748B`; surface `#FFFFFF` / bg
  `#F8FAFC`; primary `#0F766E`; border `#E2E8F0`; radius md `8px` / lg `12px`;
  space unit `4px` (common 8/12/16/24); focus ring `#14B8A6` 2px.

- **Overdue** → due chip in `--color-danger` + the word "Overdue" (not color alone).

## 9. Provenance & Known Gaps

**Provenance.** Source mode: **research** — designed from the SRS audience (§2.3:
individual private users; broad technical range) and product tone (§1.2: fast,
low-friction), grounded in productivity-app conventions. Direction "Fresh Calm"
(teal + amber) was proposed and confirmed by the user (chosen over a warmer
indigo/coral alternative). Values are designed, not extracted — treat spacing and
type scale as sound defaults open to tuning. **SRS-driven adjustments:** raw brand
teal `#14B8A6` and amber `#F59E0B` fail WCAG AA as text/button fills, so accessible
action tokens were derived (`--color-primary` #0F766E, `--color-accent-text`
#B45309, `--color-danger` #DC2626, `--color-success` #059669) while the bright
hues are retained for non-text accents.

**Known Gaps.**
- **Accessibility commitment:** built to AA, but the SRS (NFR-USE-004) commits only
  to best-practice, not formal AA conformance — confirm whether AA becomes a
  post-MVP target (SRS Appendix B, Q5).
- **Illustration/empty-state art** style not yet specified (icon-led placeholder
  assumed); pick an illustration approach before build.
- **Iconography:** Lucide chosen to pair with shadcn; not yet mapped icon-by-icon.
- **Marketing/landing** visual language is out of scope here (app + auth only).
- **Font licensing/loading** (self-host vs. Google Fonts) is an implementation
  choice; Inter assumed.
- **Exact adaptive-density breakpoint** (where rows compact) set to `lg` by
  default — validate with real content.
